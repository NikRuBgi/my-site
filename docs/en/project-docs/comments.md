# Comments & Annotations

## What are Comments?

Comments are typed annotations linked to specific entities in the building model. They are accessible from **Modeling → Comments** and from individual entity pages.

Unlike the logbook (which is building-wide), comments are attached to a specific entity and appear in its context.

## Comment Types

| Type | Use Case |
|---|---|
| Bookmark | Mark an entity or moment of interest for follow-up |
| Complaint | Record an occupant or operational complaint linked to a system |
| Defect | Document a known equipment defect or damage |
| Calibration | Record a sensor calibration event with date and value |
| Information | General note — observations, context, reference data |

## Collaboration

Comments are visible to and editable by all team members with building access. They appear in the entity's comment feed in chronological order.

Use comments for **time-stamped, entity-specific observations** — things that happened at a specific moment in the context of a specific system. Use the [Logbook](logbook.md) for broader, structural documentation.
