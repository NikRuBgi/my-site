# Implementation Guide

## What is the Implementation Guide?

The Implementation Guide is a macro-level progress tracker for the digital twin implementation. It provides a shared view of how far along the building model is across each implementation phase, and is accessible to all stakeholders involved in the project.

Navigate to **Modeling → Implementation Guide**.

## What it Tracks

The guide tracks completion status for each phase of the implementation methodology:

| Phase | Description |
|---|---|
| Building configuration | Basic setup completed |
| Energy monitoring | Sources, invoices, baseline model |
| Modeling | Entities, families, tags, and templates |
| Data acquisition | BACnet connection, point mapping, deployment |
| Analysis | Rules, KPIs, fault detection activated |

## Cooperative Space

The Implementation Guide includes a **shared collaborative space** where team members can leave notes, attach files, and communicate about implementation progress. This is distinct from the logbook — it is focused on project management rather than building knowledge.

File attachments per phase allow you to store reference documents, sign-off records, and deliverable outputs alongside the progress tracker.

## Use in Multi-Stakeholder Projects

For projects involving multiple teams (building owner, energy manager, BMS contractor, BGI analyst), the Implementation Guide provides a single point of truth for project status that all parties can access without requiring access to the technical modeling tools.
