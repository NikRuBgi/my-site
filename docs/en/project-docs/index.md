# Project Documentation — Overview

Dybee includes five built-in tools for documenting project knowledge, modeling decisions, and team communications. Good documentation is essential for long-term continuous commissioning — it preserves institutional knowledge when team members change and enables multi-stakeholder coordination.

## Documentation Tools

| Tool | Purpose |
|---|---|
| [Logbook](logbook.md) | Flexible knowledge base for modeling choices, standards, and control laws |
| [Comments & Annotations](comments.md) | Typed annotations linked to specific entities |
| Entity Notes & Files | Per-entity free text, metadata, and file attachments |
| [Implementation Guide](implementation-guide.md) | Macro-level progress tracker for the digital twin implementation |
| Tasks | Commissioning measure tracking (see [Task Workflow](../tasks/workflow.md)) |

## Entity-Level Documentation

Every entity in the model has its own documentation space:

- **Notes** — free text for observations, calibration history, and modeling notes
- **Tags** — metadata for filtering and searching across entities
- **Files** — technical plans, datasheets, maintenance records, photos

Files stored at the entity level are centralized and accessible to all team members with building access.
