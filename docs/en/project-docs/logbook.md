# Logbook

## What is the Logbook?

The logbook (Journal de Bord) is a structured knowledge base for the building project. It is accessible from **Modeling → Logbook** and is intended for information that outlives any individual task or session.

## What to Use It For

- Modeling decisions and their rationale (why a family was structured a certain way)
- Control law documentation (setpoint schedules, control sequences)
- Site-specific standards and naming conventions
- Commissioning history and major interventions
- Notes from site visits or BMS vendor meetings

## Format

The logbook supports structured text entries with headings, lists, tables, and inline images. Entries are timestamped and attributed to their author.

!!! tip
    The logbook is the right place to document anything that a new team member would need to understand the building model and analysis methodology — context that wouldn't be obvious from the model itself.
