# Continuous Commissioning

## What is Continuous Commissioning?

Continuous commissioning (CxC) is the ongoing process of monitoring building performance, detecting degradation, implementing corrective measures, and verifying results — repeated over the building's lifespan.

In Dybee, CxC is supported by the combination of:

- **Fault Detection** — automated identification of deviations
- **Tasks** — structured tracking of corrective measures from proposal to completion
- **Project Documentation** — institutional memory across team changes

## The CxC Cycle

```
Monitor → Detect → Analyze → Propose → Implement → Verify → Monitor
```

1. Rules detect a fault condition in the building model
2. The analyst investigates via the Explorer and Query Page
3. A task is created describing the problem and proposed measure
4. The task is assigned and executed by the technical team
5. Post-implementation data is reviewed to verify the correction
6. The cycle repeats

## Task Module

The Task module is the operational hub for tracking all commissioning measures. It is available on all plans and supports multi-stakeholder communication, document attachment, and financial impact tracking.
