# Task Workflow

## Creating a Task

Tasks can be created from:

- **Fault Detection** — via the "Create task" action on a fault card (pre-links the task to the entity and rule)
- **Tasks module** — manually from the task list
- **Entity pages** — directly from the entity's linked tasks section

Each task has four configuration sections:

### 1. Global Definition

| Field | Description |
|---|---|
| Task type | Category of the measure (e.g., repair, adjustment, investigation) |
| Start / due date | Timeline for the measure |
| Estimated impacts | Expected improvements: energy savings, comfort, equipment wear, GHG reduction |

### 2. Problem & Actions

- Description of the detected problem
- Proposed corrective actions
- Embedded Explorer graphics (attach trend screenshots directly)
- File attachments

### 3. Investment & Impacts

- Estimated implementation cost
- Projected energy and cost savings
- Used for prioritization and ROI tracking across the portfolio

### 4. Documentation

- Historical notes and updates
- Multi-stakeholder communication thread
- Audit trail of actions taken

## Status Workflow

```
Pending Approval → Unassigned → Assigned → In Progress → Completed → Archived
```

Tasks can also be set to **Refused** or **Deleted** if they are not actionable.

## Task Views

The task list can be viewed in three modes:

| View | Shows |
|---|---|
| Priority / Status | Default list organized by urgency and state |
| Impacts | Sorted by estimated energy or comfort impact |
| Financial Gains | Sorted by projected cost savings |

These views help prioritize which tasks to pursue when resources are limited.
